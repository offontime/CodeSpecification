## 为什么要规范代码
在团队开发的项目中，通常来讲不可能某个人一直负责某个独立模块，再加上人员的流动，通常来讲我们会要求代码使用相同的规范化语言来写。所有人遵循同样的代码规范来编写，就可以让代码具有易读、可维护、可扩展、可复用等特性：

1. 使代码易读
2. 令代码可维护
3. 段落可扩展
4. 模块可复用
5. 功能可测试
6. 简洁与稳定
